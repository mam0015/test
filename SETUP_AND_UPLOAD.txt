ALERT CONSTRUCTION — PRODUCT FOUNDATION SETUP
==============================================

This package keeps the existing black, white and yellow app design and adds:
secure accounts and team roles, cloud project sync, an editable price catalogue,
AI audit records, daylight mode, onboarding, search, confirm/undo safeguards,
and offline-capable non-AI tools.

IMPORTANT
---------
Complete the Supabase database and Edge Function setup BEFORE uploading the
website. Internal tools now require an authorised account.


PART A — CREATE THE SECURE DATABASE FOUNDATION
-----------------------------------------------

1. Open the existing Supabase project.
2. Open SQL Editor > New query.
3. Copy and run the complete file:

   supabase/migrations/20260714_product_foundation.sql

This creates:
- organisations and private team codes;
- profiles with Owner, Estimator and Site Supervisor roles;
- one shared project workspace per company;
- automatic workspace versions;
- the editable price catalogue and price history;
- AI/project audit records;
- Row Level Security policies.

Do not disable Row Level Security.

If Supabase email confirmation is enabled, a new user must confirm their email
before the first sign-in.


PART B — DEPLOY THE UPDATED AI EDGE FUNCTION
---------------------------------------------

Function name: analyse-plan
Source:
  supabase/functions/analyse-plan/index.ts

Required Edge Function secret:
  OPENAI_API_KEY = the server-side OpenAI API key

Optional secret:
  OPENAI_MODEL = gpt-5.6

Dashboard method:
1. Edge Functions > analyse-plan > Code.
2. Replace the code with the included index.ts.
3. Deploy.
4. In function settings, turn legacy Verify JWT OFF.
5. Confirm OPENAI_API_KEY exists in Edge Function Secrets.

CLI method:
  supabase functions deploy analyse-plan --no-verify-jwt

The function now requires a signed-in Supabase user, uses the authenticated
company catalogue where an office rate override exists, and never exposes the
OpenAI key to the browser.

PART D — CREATE THE FIRST OWNER AND TEAM
----------------------------------------

1. Open the deployed Dashboard.
2. Open Account at the bottom-left.
3. Choose Create Account.
4. Enter the company name and Owner email.
5. Sign in after email confirmation, if required.
6. Copy the private team code shown on the Account page.
7. Each staff member creates their own account and uses Join Team.
8. The Owner assigns each person as:
   - Owner
   - Office / Estimator
   - Site Supervisor

Owners and Estimators can edit company rates. Site Supervisors have read-only
catalogue access. AI tools, projects, calculators and site records require an
authorised account.

OFFLINE AND DAYLIGHT USE
------------------------

- Open a calculator once while online so the service worker can cache it.
- The calculators, renovation planner, saved local projects and checklist work
  without a signal after they have been cached.
- AI Plan Estimator and Quote Analysis require internet access.
- Use the bottom-left Day/Dark button for bright outdoor conditions.


PRICE CATALOGUE AND LEGACY RATES
--------------------------------

The Live Price Catalogue stores all new office overrides in the private
Supabase database and keeps a change history. It includes the existing trade
rates plus every Renovation Planner allowance, so Office/Estimator users can
update tiling, carpentry, cabinetry and other planning figures without editing
code. The planner automatically uses those saved values.

The existing Electrical, Plumbing and Cladding values are retained as offline
fallbacks so the current calculators are not broken. Because those legacy
values existed in the original static app, they are still present in the
website source. Future rate changes made through the catalogue are protected by
account access and Row Level Security.

For complete removal of every legacy fallback value from public source, first
populate the full private catalogue and then perform a separate fallback-removal
release. Do not remove the fallbacks before the private catalogue is complete.

SECURITY NOTES
--------------

- The browser contains only the Supabase publishable key. This is expected.
- No OpenAI secret or Supabase service-role key belongs in GitHub.
- AI calls require both the publishable key and a signed-in user access token.
- Customer estimates should show customer totals, not builder-rate breakdowns.
- Every AI result remains preliminary and must be checked by the Builder or
  relevant licensed trade before being issued as a quote.
